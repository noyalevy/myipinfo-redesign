What Is JSON? A Beginner's Guide to the Universal Data Format

📅 November 2024⏱ 5 min read✍️ MyIPInfo Team

JSON is the language of the modern web. Nearly every API, configuration file, and data exchange uses it. If you've ever seen curly braces with key-value pairs, you've seen JSON. Here's everything you need to understand it.

What Is JSON?

JSON (JavaScript Object Notation) is a lightweight, text-based format for storing and exchanging structured data. Despite its name, JSON is language-independent — it's used in Python, Java, Go, PHP, Ruby, and virtually every other programming language.

JSON was created by Douglas Crockford in the early 2000s as a simpler alternative to XML. Today it is the dominant data format for web APIs — when you use a weather app, a maps service, or check your IP on MyIPInfo.io, JSON is almost certainly involved in the data exchange.

JSON Syntax

JSON has two fundamental structures:

  • Object — A collection of key-value pairs wrapped in curly braces {}
  • Array — An ordered list of values wrapped in square brackets []

Here's a simple JSON object:

{
  "ip": "199.203.79.17",
  "city": "Tel Aviv",
  "country": "Israel",
  "isp": "Cellcom",
  "latitude": 32.0853,
  "longitude": 34.7818,
  "vpn": false
}

Rules to remember:

  • Keys must be strings in double quotes
  • Key-value pairs are separated by colons :
  • Pairs are separated by commas ,
  • No trailing comma after the last item (common mistake!)
  • JSON has no comments
⚠️ Most Common JSON Mistake

A trailing comma after the last item is invalid JSON and will cause parse errors: {"name": "John",}. Always validate your JSON before using it.

JSON Data Types

JSON supports exactly six data types:

{
  "string_example":  "Hello, World!",
  "number_example":  42,
  "float_example":   3.14,
  "boolean_example": true,
  "null_example":    null,
  "array_example":   [1, "two", false, null],
  "object_example":  { "nested": "object" }
}

Frequently Asked Questions

Is JSON the same as JavaScript? +
No. JSON was inspired by JavaScript object syntax, but it's a language-independent text format. JSON has stricter rules than JavaScript objects — for example, keys must be quoted strings. You can use JSON with Python, Java, Go, PHP, or any language.
How do I validate JSON? +
You can validate JSON by pasting it into our JSON Export tool, or use online validators like JSONLint.com. In code: try parsing it — if it throws a parse error, the JSON is invalid. Common errors: missing quotes on keys, trailing commas, and unescaped special characters.
What is "pretty printed" JSON? +
Pretty printed JSON is formatted with indentation and newlines to be human-readable. Minified JSON removes all whitespace to reduce file size. APIs often return minified JSON; tools like our JSON Export page display it pretty-printed for easy reading.
Can JSON contain comments? +
No — standard JSON does not support comments. This is intentional, as JSON is a data format, not a configuration language. If you need comments in config files, consider JSONC (JSON with Comments, used by VS Code), YAML, or TOML instead.
