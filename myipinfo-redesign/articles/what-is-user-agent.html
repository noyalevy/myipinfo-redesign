What Is a User Agent? How Browsers Identify Themselves

📅 March 2025⏱ 5 min read✍️ MyIPInfo Team

Every time your browser loads a webpage, it sends a "user agent" string to the server. This string identifies your browser, operating system, and device type. Here's what it means and why developers and security researchers care about it.

What Is a User Agent?

A user agent is a string of text that your browser automatically includes in every HTTP request it makes. It tells the web server what software is making the request — specifically your browser name and version, operating system, and sometimes device type.

Servers use this information to serve appropriate content. For example, a website might serve a mobile-optimized layout to an iPhone and a desktop layout to a Windows PC — all based on the user agent.

Anatomy of a User Agent String

Here's a real user agent from Google Chrome on Windows 11:

Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/124.0.0.0 Safari/537.36

Let's break it down:

  • Mozilla/5.0 — A legacy compatibility token. Almost all browsers include this.
  • (Windows NT 10.0; Win64; x64) — The operating system: Windows 10/11, 64-bit
  • AppleWebKit/537.36 — The rendering engine (Blink, which is based on WebKit)
  • (KHTML, like Gecko) — More legacy compatibility tokens
  • Chrome/124.0.0.0 — The actual browser and version
  • Safari/537.36 — Another compatibility token (Chrome impersonates Safari)
💡 Why "Mozilla/5.0" Everywhere?

This is a historical quirk. Early web servers served better content to Netscape Navigator (which identified as "Mozilla"). So every browser started including it for compatibility — a tradition that persists 30 years later.

Why User Agents Matter

For Developers

  • Responsive design testing — Emulate mobile devices in DevTools
  • Analytics — Understand what browsers and devices your users have
  • Feature detection — Serve polyfills only to browsers that need them
  • Bot detection — Web scrapers often use headless browser user agents

For Security

  • Fingerprinting — Your UA combined with other signals can uniquely identify your browser
  • Attack surface — Some vulnerabilities target specific browser versions
  • Fraud detection — Mismatched UAs can indicate bot traffic or fraud

User Agent Spoofing

You can change (spoof) your user agent to make websites think you're using a different browser or device. Common reasons:

  • Testing how your website looks on different devices
  • Accessing sites that block certain browsers
  • Scraping websites that check for browser UAs
  • Privacy — reducing your browser fingerprint

In Chrome DevTools: press F12 → Network conditions → Uncheck "Use browser default" under User agent → select or type a custom UA.

⚠️ Note on Privacy

Changing your user agent alone doesn't make you anonymous. Websites use dozens of other signals (canvas fingerprinting, fonts, screen resolution, etc.) to identify browsers. The Privacy Sandbox initiative and User-Agent Client Hints are gradually replacing the full UA string with more privacy-preserving alternatives.

Frequently Asked Questions

What is a user agent string? +
A user agent string is a text identifier that your browser sends with every web request. It tells the server what browser, operating system, and device you're using so the server can send appropriate content.
Can websites track me using my user agent? +
Your user agent is one of many signals used for browser fingerprinting. Combined with your screen resolution, installed fonts, timezone, and other factors, it can create a fairly unique profile — even without cookies.
Why does every browser say "Mozilla" in its user agent? +
It's a historical compatibility quirk from the early web browser wars. Early servers gave richer content to Netscape Navigator (Mozilla), so all browsers began including "Mozilla" to get the same treatment. It stuck as a standard prefix.
How do I change my user agent? +
In Chrome/Edge: Open DevTools (F12) → Three-dot menu → More tools → Network conditions → Uncheck "Use browser default" and set a custom user agent. Browser extensions like "User-Agent Switcher" also work.
